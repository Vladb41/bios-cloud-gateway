# 🔐 BIOS Cloud Gateway v1.1 — Инструкция по установке и настройке

## 📋 Системные требования
- Python 3.10 или выше
- Windows 10/11 или Linux (Debian 11+, Ubuntu 22.04+, macOS)
- Интернет-соединение (для запросов к YandexGPT и поиска)

## 📦 Шаг 1: Распаковка архива
- Linux: `tar -xzf BIOS_Cloud_Gateway_v1.1_Final.tar.gz && cd bios-gateway-src`
- Windows: Распакуйте ZIP-архив, откройте папку в терминале/PowerShell.

## 🔑 Шаг 2: Создание и настройка файла `.env`
Файл `.env` хранит секретные ключи. Он **не входит** в архив по соображениям безопасности.

1. Скопируйте пример конфигурации:
    cp .env.example .env

2. Получите ключи в Яндекс Облаке:
    - Войдите в console.cloud.yandex.ru
    - Создайте сервисный аккаунт с ролью `ai.languageModels.user`
    - В разделе API-ключи создайте ключ. Скопируйте Идентификатор (aje...) и Секретный ключ (AQV...)
    - На главной панели найдите Идентификатор каталога (b1g...)

3. Откройте `.env` в редакторе и вставьте:
    HOST=127.0.0.1
    PORT=5000
    YANDEX_API_KEY=AQVN...ваш_ключ...
    YC_FOLDER_ID=b1g...ваш_каталог...
    REQUEST_TIMEOUT=30
⚠️ Никогда не публикуйте `.env` в открытых репозиториях!

## 🚀 Шаг 3: Запуск
- Linux: `chmod +x start.sh && ./start.sh`
- Windows: дважды кликните `start.bat`

## 🌐 Шаг 4: Проверка
Откройте в браузере: `http://127.0.0.1:5000`
Введите вопрос. Ожидайте ответ с меткой `☁️ ✅ BIOS`.

## 🛡️ Безопасность
- Сервер работает только локально (127.0.0.1).
- Обновляйте ключи каждые 90 дней.
- `.env` автоматически игнорируется Git.

## ❓ Неполадки
- Port 5000 is in use: закройте другое приложение или смените `PORT=8080` в `.env`.
- Ошибка API: проверьте ключи и права сервисного аккаунта.
